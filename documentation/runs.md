# Runs

## Overview

A **Run** is the process of applying a **Prompt** to your tutoring data to generate **Annotations**. This is the action that triggers the LLM to begin its analysis. When you initiate a run, you define all the parameters for the analysis, including which **Sessions** to analyze, the **Annotation Type** to use, and which **Prompt** and LLM model to apply.

Sandpiper supports three types of runs:

- **LLM Runs** — The standard run type where an LLM annotates your data based on your prompt instructions.
- **Human Runs** — Created by uploading a CSV file of human-annotated labels, allowing you to compare human annotations against LLM runs in an **Evaluation**.
- **Adjudication Runs** — A special run that resolves disagreements between other runs by having an LLM vote on conflicting annotations. Adjudication runs are created from the **Evaluation** workflow.

Each run captures a **Snapshot** of the prompt and model configuration at the time it was created, ensuring full reproducibility even if the prompt is later modified.

## How to use

### Creating an LLM Run

You initiate a run from within a **Project** or a **Run Set**.

1.  **Start a New Run:** Navigate to a **Project** and click "New Run," or create runs directly from a **Run Set**.
2.  **Configure the Run:** On the run configuration screen, you will make the following selections:
    - **Sessions:** Choose the specific sessions you want to analyze.
    - **Prompt:** Select the prompt and **Prompt Version** you want to use.
    - **Annotation Type:** Choose whether to run the analysis per utterance or per session.
    - **LLM Model:** Select the LLM model you wish to use (e.g., Claude, GPT).
3.  **Review Cost Estimate:** Sandpiper will display an estimated cost based on the number of sessions, input tokens, and selected model. If your team has insufficient credits, you will be asked to acknowledge this before proceeding.
4.  **Initiate Analysis:** Once all options are set, click "Start Run." You can monitor the run's progress in real time from the **Dashboard** or the project's details page.

### Creating Human Runs

Human runs allow you to upload manually annotated data for comparison against LLM runs.

1.  **Download a Template:** From a **Run Set**, click "Download Annotation Template" to get a CSV pre-populated with your session data.
2.  **Annotate Offline:** Fill in the annotation columns in the CSV with your human labels.
3.  **Upload the CSV:** Navigate to the **Human Annotations** upload page in your run set and upload the completed CSV. Sandpiper will validate the file and create one human run per annotator.

### Managing Runs

- **Stop a Run:** You can stop a run mid-execution if needed. Sessions that have already been processed will retain their annotations.
- **Export Results:** Download the annotations from a completed run as a CSV file.
- **Add to Run Set:** Add a completed run to an existing **Run Set** for comparison with other runs.
