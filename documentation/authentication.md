# Authentication

## Overview

Sandpiper uses **GitHub OAuth** for authentication. When you sign in, Sandpiper receives your GitHub username, display name, and GitHub user ID. This information is used solely to authenticate you and manage your access to the application.

New users go through an **onboarding** flow after their first login to set up their account and join or create a **Team**.

## How to use

### Signing In

1.  **Click "Sign In":** From the Sandpiper login page, click the sign-in button.
2.  **Authorize with GitHub:** You will be redirected to GitHub to authorize Sandpiper to access your basic profile information.
3.  **Return to Sandpiper:** After authorization, you are redirected back to Sandpiper and logged in.

### Onboarding

If this is your first time using Sandpiper:

1.  **Complete Onboarding:** After your first login, you will be guided through a brief onboarding process.
2.  **Join or Create a Team:** You will be prompted to join an existing **Team** via an invite link or create a new team.
3.  **Start Working:** Once onboarding is complete, you are taken to the **Dashboard** and can begin creating projects.

### Account Security

- It is your responsibility to maintain your GitHub account security and keep your credentials confidential.
- Sandpiper does not store your GitHub password. Authentication is handled entirely through GitHub's OAuth flow.
